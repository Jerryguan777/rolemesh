RoleMesh · Kubernetes 部署网络拓扑

Rancher RKE2 · 单 namespace rolemesh · 声明式基础设施（Helm）+ verify-only 启动 · 设计稿 rev4
平台面流量（orchestrator / webui） agent 受控流量（仅两个去向） gateway 流量（白名单出网 / NATS RPC） 被 NetworkPolicy 拒绝 控制面 / 声明式动作

① agent-default-deny

selector: role=agent，Ingress + Egress 全拒、零规则。Docker Internal=true 网桥的等价物，独立存在以便启动校验单独验证"默认拒绝"不变量。

② agent-allow-egress

仅放行 agent → gateway（53 / 3001 / 3128）与 → NATS（4222）。不放行 kube-dns：域名解析只能经 gateway 白名单 resolver，DNS 防外渗。

③ gateway-policy

入站仅来自 agent / orchestrator；出站放行 NATS（4222，凭证 RPC 必需）、53/80/443 及 values 声明端口。应用层另有域名白名单，双保险。

④ orchestrator-policy

出站仅 → K8s API、Postgres、NATS、gateway、外网 HTTPS（自身 LLM 调用）。RBAC 限定为本 namespace 的 pods 增删查 watch/log。

身份模型：orchestrator 在 spawn 时签发 HMAC token 注入 agent，token 随每个请求带内出示（反代路径段 / Proxy-Authorization），gateway 用共享 Secret 无状态验签——与网络拓扑无关，Docker / K8s 两个 runtime 同一机制。  |  来源：docs/container-runtime-decoupling-design.md（rev4，2026-06-12）
